# Phase E — Upload Knowledge Base Files  
**ISOM 360 – Advising Chatbot Project**

## Purpose
To import all finalized Knowledge Base (KB) micro-chunks into the Custom GPT so the chatbot can answer accurately using university-approved content.

---

## 1. Prepare KB Micro-Chunks
Before upload, confirm:
- Files are in **.docx** format  
- Each file contains:
  - Chunk ID  
  - Title  
  - Clear student-facing explanation (150–300 words)  
  - Source URL  
  - Effective term  
  - Escalation contact  

Naming convention:
```
CategoryPrefix-ID_Title_Term.docx
Example: REG-01_AddDropBasics_Fall2025.docx
```

---

## 2. Navigate to the Knowledge Upload Panel
1. ChatGPT Plus → Explore GPTs  
2. Select your chatbot  
3. Build tab → scroll to **Knowledge**  
4. Select **Upload Files**

---

## 3. Upload All Micro-Chunks
For each advising category, upload files one by one.

### Example category structure:
| Category | Prefix | Files |
|---------|--------|-------|
| Registration | REG- | 10–12 |
| Academic Standing | AS- | 8–10 |
| Program of Study | MMC- | 8–12 |
| Residency & Double-Counting | RDC- | 5–7 |
| Holds | HLD- | 5–8 |
| Credit Load & Overload | CL- | 5–6 |

**Checklist**
- All files show green check marks  
- No duplicates or PDFs  
- File names exactly match the Chunk IDs  

---

## 4. Verify Internal Indexing
- Hover over uploaded files  
- Verify size + timestamp  
- Ensure one file per micro-chunk  

---

## 5. Repeat for All Six Categories
Upload the complete KB for all advising domains.

---

## Result
Your GPT now contains a complete Knowledge Base that provides the foundation for reliable, policy-aligned responses.
